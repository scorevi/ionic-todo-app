## 5. CRUD Implementation Details

### CREATE
**File:** `src/services/todoService.ts` - `createTodo()`
- Takes title, description, priority, dueDate, and category as parameters
- Priority defaults to "medium", category defaults to "General"
- Adds new document to Firestore "todos" collection
- Sets completed to false by default
- Adds createdAt and updatedAt timestamps
- Returns the document ID

### READ
**File:** `src/services/todoService.ts` - `getAllTodos()` and `getCompletedTodos()`
- Queries Firestore collection
- Orders by createdAt (newest first)
- Filters by completed status for completed tab
- Maps Firestore documents to Todo objects with all fields (priority, category, dueDate)
- Provides default values for missing fields (priority: "medium", category: "General")
- Returns array of todos

### UPDATE
**File:** `src/services/todoService.ts` - `updateTodo()` and `toggleTodoComplete()`
- Takes document ID and updated fields (title, description, priority, category, dueDate)
- Updates specific document in Firestore
- Updates updatedAt timestamp automatically
- Checkbox toggle updates only completed status
- Edit modal updates all fields including priority, category, and due date

### DELETE
**File:** `src/services/todoService.ts` - `deleteTodo()`
- Takes document ID as parameter
- Permanently removes document from Firestore
- Triggered by swipe-left gesture and trash button
